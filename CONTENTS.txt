🎯 QUICK REFERENCE
────────────────────────────────────────────────────────────────────────────────

File Organization:
  include/     → Model/data/loss interface definitions
  src/         → Implementation files (one .cpp per component)
  *.md         → Documentation (one per topic)
  CMakeLists.txt, build.sh → Build system

Executable Build Order:
  1. train_restoration  - Training (2 hours compile + 10+ hours run)
  2. restore_images     - Inference (2 hours compile, 40ms per image run)
  3. evaluate          - Metrics (2 hours compile, 5 seconds run)
  4. visualize         - Visualization (2 hours compile, fast)

Key Files to Edit:
  • include/config.hpp → Hyperparameters (batch size, learning rate, etc)
  • include/model.hpp → Architecture changes
  • src/train.cpp → Training logic modifications
  • src/dataloader.cpp → Custom preprocessing

Training Path:
  1. Edit include/config.hpp (select configuration)
  2. mkdir build && cd build
  3. cmake -DCMAKE_PREFIX_PATH=/path/to/libtorch ..
  4. cmake --build .
  5. ./train_restoration (wait 10-12 hours)
  6. ./restore_images data/EUVP/test/murky output
  7. ./evaluate data/EUVP/test/murky data/EUVP/test/reference \
              checkpoints/generator_A2B_epoch_200.pt results.csv
